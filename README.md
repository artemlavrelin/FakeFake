# 🔥 ContestBot v4

Telegram-бот для конкурсов в закрытом комьюнити.
**Личка** — весь функционал. **Группа** — витрина и анонсы.

---

## 🚀 Быстрый старт (локально)

```bash
cd contest_bot_v4
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env       # заполни переменные
python bot.py
```

---

## ⚙️ Переменные окружения

| Переменная | Обязательно | Описание |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен от @BotFather |
| `ADMIN_IDS` | ✅ | ID администраторов через запятую |
| `DATABASE_URL` | ✅ | SQLite или PostgreSQL |
| `GROUP_ID` | рекомендуется | Числовой ID группы (бот — администратор) |
| `BOT_USERNAME` | рекомендуется | Username бота без @ (для deep-link кнопки) |
| `REDIS_URL` | нет | FSM Redis (иначе MemoryStorage) |
| `WEBHOOK_HOST` | нет | Домен Railway — включает webhook-режим |
| `PORT` | нет | Порт сервера (Railway задаёт автоматически) |
| `ATM_TEXT` | нет | Текст раздела 🧲 ATM |
| `LOG_TO_FILE` | нет | `1` — писать логи в `logs/contest_bot.log` |

---

## 🚂 Деплой на Railway

1. **GitHub** → подключи репозиторий в Railway
2. **+ New → PostgreSQL** → Railway добавит `DATABASE_URL` сам
3. **+ New → Redis** → Railway добавит `REDIS_URL` сам
4. **Variables** → добавь вручную:
   ```
   BOT_TOKEN=...
   ADMIN_IDS=...
   GROUP_ID=...
   BOT_USERNAME=...
   WEBHOOK_HOST=https://your-project.up.railway.app
   ```
5. **Settings → Networking → Generate Domain** → скопируй URL → вставь в `WEBHOOK_HOST`
6. Пуш в репо → Railway запускает бота через `railway.toml`

> Бот должен быть **администратором** группы (права: читать сообщения, закреплять сообщения).

## 🎮 Пользовательский сценарий

```
/start
  └─▶ Главное меню
        ├─ ⚡️ Розыгрыш          → экран конкурса (#N, описание, stats bar)
        │     └─ 🔥 Участвовать  → экран подтверждения
        │           └─ ✅ Подтвердить → зарегистрирован
        ├─ 🔥 Участвовать        → тот же экран конкурса (шорткат)
        ├─ 📱 Моя статистика     → участий / побед / сумма / дата
        ├─ 🌍 Общая статистика   → публичные цифры + топы
        │     ├─ 🏆 Топ победителей
        │     └─ 👥 Топ участников
        └─ 🧲 ATM               → статический информационный экран
```

### Кнопка в группе
Закреплённое сообщение с кнопкой **🔥 Участвовать** открывает бота через deep-link и сразу показывает экран конкурса.

## 🔧 Админ-панель (`/admin`)

| Действие | Описание |
|---|---|
| ⚡️ Создать конкурс | FSM 4 шага: описание → приз (текст) → приз (сумма) → победители |
| ✏️ Редактировать | Изменить любое поле активного конкурса |
| 🚫 Отменить | Досрочная отмена с подтверждением |
| 🎲 Розыгрыш | Рандомный выбор победителей + уведомление всех участников |
| 📣 Рассылка | Массовая рассылка с превью и подтверждением |
| 👥 Пользователи | Список всех зарегистрированных |
| `/ban <id>` | Заблокировать пользователя |
| `/unban <id>` | Разблокировать |

## 🛡️ Защиты

| Слой | Механизм |
|---|---|
| Только личка | `PrivateChatOnlyMiddleware` — первый в цепочке |
| Спам кнопок | `ThrottleMiddleware` — 2 сек между одинаковыми callback |
| Двойное участие | Проверка `is_participant` на шаге показа и на шаге записи в БД |
| Бан | Проверка `is_banned` на обоих шагах участия |
| Один конкурс | Нельзя создать новый пока есть активный |
| Fail-safe draw | `min(winners_count, participant_count)` — никогда не упадёт |

## 📬 Уведомления после розыгрыша

- **Победители** → персональное поздравление с информацией о призе
- **Остальные** → список победителей + благодарность
- **Группа** → публичный анонс результатов
- Задержка 50ms между отправками (≈20 msg/sec, в рамках Telegram-лимита)
